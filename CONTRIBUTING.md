# Contributing

Thank you for helping improve this community resource. This repository is a Markdown-only curated list, not a data mirror or software platform.

## What Kinds of Resources Are Welcome

Relevant resources include:

- Holocaust testimony and oral-history datasets, archives, portals, and catalogs.
- Transcripts, subtitles, TEI/XML editions, and interview metadata.
- ASR, speech, video, and multimodal resources.
- NER, entity-linking, sentiment, emotion, topic, and narrative-analysis resources.
- Vocabularies, gazetteers, authority files, and thesauri.
- Tools and models directly useful for oral-history AI/NLP research.
- Responsible AI, access-control, RAG, and evaluation resources tied to sensitive testimony or oral-history material.

Please do not add generic NLP resources unless their relevance to oral history, testimony, archives, cultural heritage, or sensitive narrative collections is clear.

## Required Information for New Resources

Each proposed main-list entry should include:

- Resource name.
- Official URL.
- Maintainer or institution.
- Short description.
- Modality.
- Languages.
- Access level.
- License or terms.
- Relevant purposes or tasks.
- Suggested KI.OH or small-model use.
- Notes on ethical, legal, or access constraints.

If any of this information is unclear, add the resource to `Needs Verification` instead of the main curated list.

## How to Classify Access

Use the access legend from the README:

- 🟢 Open download: direct download under stated terms.
- 🔵 Public web access: searchable or viewable online, but bulk download may not be available.
- 🟡 Metadata only: records, descriptions, dashboards, or metadata are public, but full content is not.
- 🟠 Registration required: free account, registration, or request needed.
- 🔴 Institutional access: subscribing institution, reading room, access site, or approved network needed.
- ⚫ Licensed/restricted: explicit permission, data agreement, or special license needed.
- ❓ Needs verification: public availability, license, or scope is not confirmed.

Do not call something open unless there is a clear download path and license or terms.

## How to Classify Modality

Use the README modality labels:

- Text / Transcripts.
- TEI / XML.
- Audio.
- Video.
- Multimodal.
- Images / Scans.
- Metadata / Catalog.
- Annotations.
- Vocabularies / Authority Data.
- Models / Tools.

Use multiple labels when needed, but keep entries readable.

## How to Classify Purpose

Choose the most relevant tasks from the README browse categories, such as:

- Semantic search and retrieval.
- Retrieval-augmented generation.
- Ask-the-archivist or research assistant design.
- Automatic speech recognition.
- Speech disfluency, pauses, and paraverbal cues.
- Named entity recognition.
- Entity linking and authority control.
- Multilingual and cross-lingual retrieval.
- Yiddish and low-resource language processing.
- Sentiment, emotion, and narrative trajectories.
- Topic modeling and thematic indexing.
- OCR and historical scanned documents.
- Video and multimodal analysis.
- Metadata, catalog search, and archive discovery.
- Access-control-aware search.
- Small-model evaluation.
- Responsible AI, bias, and hallucination testing.

## Sensitive-Data Policy

Do not contribute:

- Restricted testimony data.
- Scraped testimony content.
- Personal data copied from archives.
- Mirrors of protected audio, video, transcripts, or documents.
- Model-training recipes that assume unauthorized reuse of restricted collections.

Respect archive terms, consent contexts, survivor and interviewee dignity, and community expectations. When in doubt, link to official metadata or documentation rather than copying content.

## Verification Checklist

Before adding a resource to the main list, check:

- Is the URL official or institutionally reliable?
- Is the resource directly relevant to this list?
- Is the access status clear?
- Is the license or terms-of-use page linked or summarized?
- Are language coverage and formats supported by public documentation?
- Is the description neutral and non-promotional?
- Does the entry avoid personal data and protected content?
- Should it go to the main list or `Needs Verification`?

## Style Guide

- Use concise descriptions.
- Do not overclaim.
- Prefer public source language over speculation.
- Use `unknown`, `access status unclear`, or `needs verification` when uncertain.
- Prefer `public web access` or `metadata only` when full download is not clearly available.
- Keep resource cards consistent with the README template.
- Add one canonical resource entry, then link to it from browse indexes.
- Do not quote non-public notes, correspondence, restricted documents, or project materials.
